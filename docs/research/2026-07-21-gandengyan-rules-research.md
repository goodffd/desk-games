# 干瞪眼规则调研笔记

> 调研日期：2026-07-21（北京时间）
> 用途：为 desk-games 第三个游戏模块「干瞪眼」钉死可实现规则集。engine 是规则唯一真相，本笔记的分歧清单即 `RuleConfig` 开关候选。
> 调研范围：平台官方规则页 8 家、GitHub/Gitee 开源实现 19 个真实含规则代码的仓库、英文牌类权威站 3 家、中文次级来源 20+。

---

## 0. 先说三条会推翻既有假设的结论

**① 干瞪眼不是「跑得快变体」，核心机制是「大一法则」。**
跟牌必须是**同牌型、同张数、点数正好大一级**——上家出 5，你有 A、K、Q 也只能干瞪眼，只能出 6（或 2、或炸弹）。这条机制在 **9 个互相独立的代码库**里被硬编码，在 pagat.com、腾讯 QQ 游戏、JJ 比赛、91y、边锋、17dp 的规则页上逐字出现。这与斗地主/跑得快「出更大的就行」是根本不同的博弈，AI 搜索空间、手牌评估函数全部不能从掼蛋那边照搬。

**② 顺子最短 3 张，不是 5 张。**
全部 19 个开源仓库的顺子校验里**找不到任何一个 5 张长度常量**；全部平台规则页写的都是「三张点数相连」。如果按想当然的「顺子最少 5 张」写 engine，一行代码就把整个游戏做错了。

**③ 任务书里「河南/陕西 + 憋七近亲 + 干扎」这套前提，查下来基本不成立。**

| 原假设 | 调研结果 |
|---|---|
| 主流变体在河南、陕西 | 「河南干瞪眼」「山西干瞪眼」「关中干瞪眼」精确检索**零结果**；「陕西干瞪眼」「湖北干瞪眼」**只命中同一个 SEO 空壳页**（[zadiqp.com](https://www.zadiqp.com/sxgdywfgz.html)，正文无规则内容）。起源口径一致指向**四川/西南**：pagat「originated during the early- to mid-2010s in Shudi in the Sichuan province」、catsatcards「Southwest China, around 2015」、[百度百科](https://baike.baidu.com/item/%E5%B9%B2%E7%9E%AA%E7%9C%BC/63808)「起源于四川，流行于湖北和四川」。真实存在地理落点的民间版本是**四川**和**山东青岛**。陕西话确实高频说「干瞪眼」，但那是[方言成语义](http://jingbianrenda.gov.cn/xxsd/dfctwh/1392038304154234881.html)（「束手无策叫干瞪眼」），不是牌名证据。 |
| 是「憋七」的近亲 | **完全无关**。憋七＝排七＝接龙，同花色顺序接龙+扣牌憋人，[百度百科](https://baike.baidu.com/item/%E6%86%8B%E4%B8%83/9926625)、[维基·排七](https://zh.wikipedia.org/zh-hans/%E6%8E%92%E4%B8%83)、QQ 游戏库三方一致，且 QQ 把干瞪眼(game/137)与憋七(game/147)列为两个独立游戏。真正容易混的是**信阳黑七**（升级类）和**憋死牛**（民间棋类，《中国民间游戏总汇》编在「憋棋类」下）。 |
| 「干扎」是别名 | 「干扎」「干炸」作为玩法名**精确检索零命中**，建议直接废弃这个写法。「干瞪眼儿」只是北方儿化口语，无独立产品。 |
| 有「去掉 3456 等小牌」的版本 | **全网零实据**。去小牌是**跑得快**的特征（[长沙随便玩官方](https://www.91sui.com/Home/GameDetail?id=2)：去大小鬼+3 个 2+黑桃 A＝48 张、每人 16 张、顺子 5 张起）。不要照着实现。 |

**④ 一个命名陷阱**：**《仙桃干瞪眼》是麻将不是扑克**——[江汉游官方](http://www.1717520.com/gamedetail.do?gameId=159)原文「仙桃干瞪眼（癞晃）起源于广东推倒胡麻将……有万、筒、条共 108 张；可以碰，可以笑」。搜「湖北干瞪眼」时极易踩到。

## 1. 变体谱系总览

核心判断：**这不是一个游戏，是一个规则族。**「标准干瞪眼规则」这种东西不存在——同一个平台自家两篇官方页都能互相打脸（91y、同城游各有实例）。

```
干瞪眼（出牌类 shedding/climbing，核心＝大一法则 + 小手牌 + 摸牌堆）
├── A 四川原生系（"关瞪瞪"派）        起源地，54 张，王当听用，三张即炸
│   └── A-2 同城游 d9613 异端支        108 张，带三带一/姐妹对，"123 顺为最小"
├── B 山东青岛酒桌系                   混子+天王炸，三带一/四带二，掀引牌，喝酒罚则
│   ├── 胶南/黄岛：埋地雷
│   └── 城阳/即墨：轮转癞子（首局 3 当混子，次局 4，依次类推）
├── C 上海系                           "2 断牌改牌"——2 不是最大单张而是牌型重置键【孤证】
└── D 平台工业化系（各家自成一套，彼此不兼容）
    ├── D-1 JJ 比赛/竞技世界   ★离主流最远：去王+6 张赖子=58 张、固定 3 人、有出必出禁过牌
    ├── D-2 腾讯 QQ 游戏       54 张，三张是普通牌型不是炸，明确允许过牌
    ├── D-3 91y               54 张，三张=炸/四张=氢弹/双王最大（站内三页自相矛盾）
    ├── D-4 边锋·游戏茶苑      连对叫"板凳"，软炸=带鬼炸（与 91y 定义相反），报警不输赢
    ├── D-5 17dp 弈棋耍大牌    唯一把"全员摸/仅赢家摸"做成两种并存模式
    ├── D-6 阿拉游·人民棋牌    唯一处理癞子牌型歧义（弹窗让玩家选）；三张算不算炸取决于副数
    ├── D-7 腾讯心悦          三张直接叫"软炸"；双王不能同时参与同一手牌
    └── D-8 同城游            站内四套互斥，整体存疑，只可作"存在此说"的证据
```

**英文权威站给的是另一条独立证据链**（未受中文 SEO 通稿污染，价值高）：
[pagat.com](https://www.pagat.com/climbing/gandengyan.html)、[gamerules.com](https://gamerules.com/rules/gan-deng-yan/)、[catsatcards](https://www.catsatcards.com/Games/Gan-Deng-Yan.html) 三家独立描述与中文主流高度吻合（54 张、庄 6 闲 5、摸牌堆、大一法则、顺子 3 张起、2 不入顺、三张即炸、双王最大、王是百搭不能单出、按剩牌张数计分），且 pagat 的 Variations 一节系统列出了 8 条并列做法，是本次分歧清单的重要来源。

### 名字由来：四个版本，其中两个主客颠倒

| 版本 | 主张 | 出处 |
|---|---|---|
| A 自己接不上 | 「下家没有可以接续上家出牌的顺牌时，只能眼巴巴地看着」 | [百度百科](https://baike.baidu.com/item/%E5%B9%B2%E7%9E%AA%E7%9C%BC/63808) |
| B 明明大得起，因越级而不许接 | 「即使手中的牌点数比较大也不能接牌，只能'干瞪眼'」 | [QQ 游戏官方](https://qqgame.qq.com/game/137.shtml)；开源 README「你有 AKQ 也只能干瞪眼」 |
| C 是让**对手**干瞪眼 | 「在你能出牌的时候，对手无法出牌」 | [App Store](https://apps.apple.com/us/app/id1554281051)、[百科·欢乐干瞪眼](https://baike.baidu.com/item/%E6%AC%A2%E4%B9%90%E5%B9%B2%E7%9E%AA%E7%9C%BC/746557) |
| D 语源先行，游戏借用俗语 | 汉典「形容空着急而无可奈何」，《醒世姻缘传》已有用例 | [汉典](https://www.zdic.net/hans/%E5%B9%B2%E7%9E%AA%E7%9C%BC)、[91y 官方](https://m.91y.com/help/guize/340.html) |

**任务书猜测的「名字来自不能主动过牌」——没有任何来源支持。** 四个版本全部指向「接不上/压不住」，即大一法则本身。

### 2.2 牌型与压制

| 项 | 主流做法 | 反例/并列做法 |
|---|---|---|
| 单张 | 3<4<…<K<A<2，**2 最大** | 全体一致 |
| 对子 | 同点两张，**对 2 压任意对子** | 全体一致 |
| 顺子最短 | **3 张** | 无反例（19/19 仓库 + 全部平台） |
| 顺子里的 A/2 | A 只当高位，顺子 3→A 封顶；**2 不入顺** | 同城游 d9613「**123 顺为较小**」✓【孤证】；开源 tobughan 允许 K-A-2；开源 kurkruk 允许 A-2-3 环绕并排为最小顺子 |
| 顺子最长 | 不限 | pagat 变体：**有些限最多 3 个点数**（5-6-7 合法，5-6-7-8 不合法）✓ |
| 连对 | **2 对起**（3344→4455） | **3 对起**：houjoe✓、百度经验「最小连对 334455」、开源 elisaday |
| 三张 | 多数当**最小炸弹**（软炸） | **QQ 游戏 + 百度百科**：普通牌型，222 最大可接任意三张，否则要顺大；**阿拉游**：一副牌时算炸、两副牌时不算 ✓ |
| 三带一/四带二/姐妹对 | 主流**没有** | 山东青岛版 ✓ + 同城游 d9613 ✓（两独立来源，少数派） |
| 三顺/飞机 | 主流**没有**（19 个仓库无一实现） | pagat 变体：三张不当炸时可有三顺 ✓ |
| 炸弹 | 3 张起，张数越多越大 | JJ **分级到 10 张炸** ✓；同城游/elisaday **炸弹＝4 张封顶** |
| 王炸 | 双王最大，不可被压 | JJ **无王故无王炸**；纯赖子派**王炸不存在** |
| 炸弹压任意牌型 | 是 | 开源 LinneyRUNN（四川）：**出了 2 之后连炸弹都压不住** |
| 癞子 | 大小王当百搭，**不能单出** | JJ 用 **6 张专用赖子牌**；erizhang/tobughan 加「广告牌」；山东城阳/即墨**轮转癞子**（每局换点数） |
| 王能否当 2 | 多数**不能** | 阿拉游：「2 不能和听用组成对牌」；[知乎专栏](https://zhuanlan.zhihu.com/p/598064403)承认「有些规则可以充当 2」；pagat 把「不许王当 2」列为变体（即默认可以）✓ |

### 2.3 流程与结算

| 项 | 并列做法 |
|---|---|
| 首攻 | 自由出任意牌型（绝大多数）｜不许先出炸弹（喜马拉雅【孤证】）｜必须由持最小牌者发起且须含该张（开源 elisaday【孤证，与所有来源冲突】） |
| 首攻能否过牌 | **不能**（源码一致：`cannot_pass_first_play` / `TABLE_EMPTY_CANNOT_PASS` / `'首出不能过牌'`） |
| 要得起能否过牌 | **能**：QQ✓「或选择不出」、91y✓「或者选择不出」、pagat✓「If you cannot play **or choose not to play**, you must pass」、全部源码｜**不能**：**JJ 两页**✓「有出必出……有能管得上的必须出牌，不允许过牌」｜措辞倾向强制：百度百科✓「下家必须打出与上家相同数量的顺牌，或炸弹」｜[CSDN](https://blog.csdn.net/mchdba/article/details/113733612) 明说两套并存 |
| 摸牌触发 | **全员各摸 1 张**：JJ✓、QQ✓、91y 帮助页✓、边锋✓、心悦、同城游✓、17dp 默认✓、PTT✓｜**仅该轮胜者摸 1 张**：91y 心得页✓、17dp「单人补牌模式」✓、百度经验、阿拉游✓、开源 jilv220/coolkid98/erizhang 等 6 家、简书上海版 |
| 摸牌顺序 | 从上轮最后出牌者开始逆时针（pagat✓、JJ✓「当圈最大者最先抓」）——**牌不够分时顺序决定谁摸得到** |
| 牌堆摸完 | 口径一致：不再摸，凭手牌打到底（91y✓、同城游✓、pagat✓「remaining players do not draw」） |
| 只剩一张王 | 只能过、等局终（91y✓、边锋✓、百科✓）｜**pagat**✓：轮到你领出且只剩一张王时，**可以打出并立即获胜**（并列变体：有些不许） |
| 结束条件 | 出完手牌即胜 | pagat 变体：**牌堆剩 >5 张时不算赢**，继续打 ✓ |
| 计分骨架 | 输家＝底分 × **剩余张数** × 倍数；赢家＝输家之和。**全部来源一律按张数，无一按牌面点数** |
| 炸弹倍率 | JJ✓ 每炸一律 ×2 连乘｜91y 帮助页✓ 炸×2/氢弹×4/双王×8/春天×2/豪春×8｜91y 心得页✓ 氢弹与双王同为 ×4｜边锋✓ 软炸(带鬼)×2/硬炸(纯)×4/双王×4｜心悦 软炸(三张)×2/炸弹×4/王炸×8｜houjoe✓ **线性累加**（炸一次×2、两次×3）｜pagat✓ 三张×2/四张×4/双王×8 |
| 个人倍数 | 17dp✓/QQ✓：剩炸弹×2、剩王×2、剩 2×2、**剩牌 ≥10 张 ×3**｜pagat✓：**每张 A ×2**｜百科 9230414✓：剩王按张数翻倍 |
| 春天/关门 | 一张未出（多数）｜**剩 5 张就算**（91y 心得页✓、百度经验✓）｜要求全桌都没出（百科✓）｜庄家只出第一手也算（边锋✓、JJ✓） |
| 特殊结算 | JJ 疯狂版**放走包赔**（领出方手牌不含 2/赖子/炸弹 + 不全是单牌 + 下家报单）✓｜边锋**报警状态（剩 1 张）不产生输赢**✓｜91y **豪春 ×8**（一把出完）✓｜**天胡 ×2**（庄家一手出完 6 张）✓｜JJ 疯狂版**宝牌**十档 ×2~×50✓｜pagat✓ 只记赢家正分、输家不扣分 |

## 4. 分歧清单（需要拍板）

每条给出并列做法、出处、以及**不拍板会导致什么后果**。编号 D1–D23，建议直接按编号回复决定。

### D1 ★★★ 要得起能不能主动过牌（`mustPlayIfAble`）
- **能过**：[QQ 游戏](https://qqgame.qq.com/app/gamedetail_137.shtml)「必须出比上家大的牌型**或选择不出**」；[91y 帮助中心](https://m.91y.com/help/guize/340.html)「或者选择不出」；[pagat](https://www.pagat.com/climbing/gandengyan.html)「If you cannot play **or choose not to play**, you must pass」；**全部开源实现**
- **不能过（有出必出）**：[JJ 标准版](https://www.jj.cn/help/news/new_85165.shtml)、[JJ 疯狂版](https://www.jj.cn/help/news/new_85166.shtml)「游戏采用**有出必出**，跟牌时，有能管得上的必须出牌，**不允许过牌**」，并附反例说明
- **倾向强制但未说死**：百度百科「下家**必须**打出与上家相同数量的顺牌，或炸弹」
- **后果**：这不是参数微调，是**两个不同的游戏**。禁过牌会彻底删掉「留牌做局/憋炸弹」这一整层策略，AI 的整个决策树都不一样。JJ 把它写进主要规则第 4 条，说明竞技化实现是刻意钉死的。
- **我的判断**：默认「能过」（证据压倒性多数 + 与名字由来的四个解释都不冲突），把「有出必出」做成配置项。但这条必须你点头。

### D2 ★★★ 三张同点算不算炸弹（`tripleIsBomb`）
- **算炸**：JJ✓、91y✓、心悦✓、边锋✓、pagat✓、山东版✓、9 个开源仓库
- **不算，是普通牌型**：[QQ 游戏](https://qqgame.qq.com/app/gamedetail_137.shtml)「三张点数大的就可以大过」（甚至**不受大一限制**）、百度百科、开源 cdzwm/elisaday
- **取决于副数**：[阿拉游](http://www.alayou.net/gamedetail.do?gameId=96)「两副牌中 3 连为普通牌型，不为炸弹，4 连以上为炸弹」
- **后果**：三张是炸的话，5 张手牌里出现炸弹的概率极高，游戏节奏完全不同；且它连带决定「三顺/飞机」这个牌型存不存在。

### D3 ★★★ 一轮无人接牌后谁摸牌（`drawMode`）
- **全员各摸 1 张**：JJ✓、QQ✓、91y 帮助页✓、边锋✓、心悦、同城游✓、17dp 默认✓、pagat✓
- **仅该轮胜者摸 1 张**：91y 心得页✓（同站自相矛盾）、百度经验、阿拉游✓、开源 6 家、简书上海版
- **17dp 直接把它做成两种模式并存**：「**单人补牌的干瞪眼规则**为：当一轮出牌结束，只有出牌方有权摸一张牌」
- **旁证争议本身**：华课网校有篇文章标题就叫《干瞪眼玩法每人都要摸牌吗》——提问即争议信号
- **后果**：只摸一张 → 牌堆消耗慢、赢家滚雪球；全桌摸 → 落后方能续命。这条直接决定一局的长度和胜负分布，**AI 的手牌规划完全不同**。

### D4 ★★ 炸弹之间怎么比
四种互斥做法：
1. **纯张数**：JJ✓「10张炸>9张炸>…>3张炸」，点数完全不参与
2. **点数优先 + 张数破例**：91y✓「点数大的就可以大过，但四张炸（氢弹）大过三张带王的炸弹（"5555"打过"JJJ王"）」
3. **张数优先，同张数不比点数（可以不顺炸）**：边锋✓、开源 erizhang✓
4. **炸弹也要顺大**：百度百科✓「上家出四个 3，则下家必须出四个 4 或更大」
5. **额外维度**：**纯炸 vs 癞子炸**——山东版✓「混子炸 < 纯数炸 < 天王炸」；[贴吧玩家原声](https://tieba.baidu.com/mo/q/newtopic/topicTemplate?topic_id=3039316)「纯的比混子大」；开源 herestars 源码有「上一次是软炸，这次是硬炸」分支

### D5 ★★ 「软炸/硬炸」定义两家完全相反
- **按张数分**：91y 术语页✓「硬炸＝4 张，软炸＝3 张」；心悦✓「软炸(三张)」
- **按含不含癞子分**：[边锋·游戏茶苑](http://www.5158wan.com/news/201807/7111.html)✓「加上'鬼'牌凑成的炸弹称为**软炸**，三张或四张相同的牌称为**硬炸**」
- **后果**：这是**术语陷阱**，不只是规则分歧。UI 上打「软炸」两个字，两地玩家理解相反。建议 engine 内部类型名**不要用「软炸/硬炸」**，用 `bombSize` + `containsWild` 两个正交字段。

### D6 ★★ 牌库构成
54（主流）｜108 两副（同城游、百科的 3 人以上、pagat 变体）｜**58＝去双王+6 张赖子**（JJ）｜**55＝54+广告牌**（开源 erizhang/tobughan）｜52 去王（同城游 d28001/d28225、开源四川版）
- pagat 双副变体有个 engine 必须处理的细节✓：「两副牌时需要**一红一黑**两张王才构成王炸，两张同色王不成炸」，且「炸弹最多 4 张同点或 2 张王，**没有 5 条或三王炸**」

### D7 ★★ 人数与起手张数
庄 6 / 闲 5（压倒性多数）｜庄 7 / 闲 6（百度百科【孤证且自相矛盾】+ 开源 jilv220）｜每人 5 张不加张（阿拉游、简书上海版、开源 elisaday）｜每人 5 张 + 掀第 6 张当引牌（山东青岛版）
人数：2–4 / 2–5 / 2–6 / 3–5 / **固定 3 人（JJ）** / 2–8（开源 erizhang）

### D8 ★ 连对最短几对
**2 对**：91y✓、17dp✓、JJ✓、QQ✓、同城游✓、pagat✓｜**3 对**：houjoe✓、百度经验「最小连对 334455」、开源 elisaday

### D9 ★★ 癞子流派：王能不能既当百搭又组王炸
- **纯赖子派**：houjoe✓明说「选了这个之后，**王炸组合就不存在了**」；JJ 干脆去掉王换成 6 张专用赖子牌
- **王炸派**：绝大多数平台——王既可百搭又可双王成炸（逻辑上自相矛盾但普遍存在）
- **心悦独有限制**✓：「**两张王不能同时与其他牌一起出**」
- **王能否代替 2**：多数不能；pagat 把「不许王当 2」列为**变体**（意味着 pagat 的默认是**可以**）✓；阿拉游「2 不能和听用组成对牌」

### D10 ★★ 癞子牌型歧义谁来消解
**全网只有[阿拉游](http://www.alayou.net/gamedetail.do?gameId=96)一家写进规则**✓：「如果牌型模糊，必须由玩家选择其中的一种牌型来出，如：王+5+6，弹出对话框选择：**4,5,6 还是 5,6,7？**」
- 开源 cdzwm 的做法是 `getSuits()` **返回所有合法解释的数组**，交给上层决策
- **后果**：这是掼蛋「逢人配」踩过的同一个坑的加强版——干瞪眼里因为有大一法则，**同一手牌选哪种解释直接决定下家能不能接**，不是等价选择。必须在协议层设计成「出牌时携带显式解释」，不能让 engine 事后推断。

### D11 ★ 顺子/连对有没有长度上限
不限（主流）｜**pagat 变体**✓：「有些不允许顺子或连对超过三个点数，即 5-6-7 与 5-5-6-6-7-7 合法，5-6-7-8 与 5-5-6-6-7-7-8-8 不合法」

### D12 ★★ 计分：线性加还是指数翻，个人倍数算不算
- 线性：houjoe✓「炸一次×2、炸两次×3」；开源 tobughan「剩牌 × (1 + 总炸级)」
- 指数：JJ✓「倍数持续相乘」；同城游「4 个炸弹 → 16 倍」；开源 rys-123/kurkruk `Math.pow(2, bombs)`
- 分种类给不同倍率：91y✓ / 边锋✓ / 心悦（三家互不相同，见 §2.3 表）
- 叠个人倍数：17dp✓/QQ✓（剩炸弹/王/2/≥10 张再乘）；pagat✓（每张 A ×2）
- 只记赢家正分：pagat 文山版✓「no negative scores for the losers」
- 上限：开源 V1ctor2182 **封顶 100 倍**
- **后果**：这是「最容易当场翻脸」的一条，结果能差一个数量级。

### D13 ★ 春天/关门的定义
一张未出｜剩 5 张就算｜要求全桌都没出｜庄家只出第一手也算。四种并存，且**同一站点自相矛盾**（91y 心得页 vs 术语页）。术语还有五种叫法：关死 / 关牌 / 被关 / **关瞪瞪**（四川）/ 春天。

### D14 ★ 平台自定义特殊结算（要不要抄）
JJ 疯狂版**放走包赔**✓｜边锋**报警状态不输赢**✓｜91y **豪春 ×8**✓｜**天胡 ×2**✓｜JJ **宝牌**十档✓｜JJ 疯狂版**随机初始倍数 10~500**✓。这些**没有任何两家重合**，全是各平台商业设计，不是民间规则。

### D15 ★★ 「大一」还是「可平可大」
pagat 明确列出变体✓：「有些允许普通牌型被**同点数**或**大一级**的牌跟——6 可以被 7 跟，也可以被另一张 6 跟；4-4 可以被 4-4 或 5-5 跟」。
中文来源无一提及此变体。**这是纯英文来源的孤证**，但它动的是核心机制，必须知道它存在。

### D16 ★ 最后一张不能留 2 / 只剩一张王
- 「最后一张不能留 2」：山东青岛版✓「想胜利的这人**最后一张牌不能留 2**，你只能出 2 或者选择不上牌」+ 开源 tobughan「手剩 2 张含 2 时必须先出 2」（**两处独立来源**，非孤证）
- 只剩一张王：只能过等局终（91y✓、边锋✓、百科✓）vs pagat 变体可出并赢✓

### D17 ★ 牌堆没摸完能不能赢
**pagat 变体**✓：「有些规定牌堆剩超过 5 张时不能赢——此时出完牌的玩家也要在本轮结束时摸牌继续打」。中文来源无一提及。

### D18 ★ 报单义务与罚则
所有来源都只写「剩 1 张必须报单」，**没有一条写明不报单罚什么**。这是民间自行约定、最容易临场吵的空白。pagat 变体：「有些要求手牌剩 3 张或更少时报出」✓（阈值不是 1 而是 3）。

### D19 ★ 要不要三带一/四带二/姐妹对
只有山东青岛版✓和同城游 d9613✓两家有（两独立来源），主流 19 个仓库无一实现。**注意**：同城游 d9613 我方判断很可能是把跑得快/争上游的规则文误贴成干瞪眼——但这只是判断，不是来源明说。

### D20 ★ 轮转癞子
山东城阳/即墨版✓：「除大小王为混子外，**第一局牌一般从 3 开始为混子，第二局 4 为混子，以此类推**。还有的地方……炸弹的规矩定为三张牌为炸」。engine 要覆盖这支，癞子点数必须是**每局可变参数**，不能写成常量。

### D21 ★ 出了 2 之后还能不能炸
开源 [LinneyRUNN/sichuangangdenyan](https://github.com/LinneyRUNN/sichuangangdenyan)（四川）源码注释：「**规则1：如果上次出的是 2，不能被任何牌接**」——炸弹也不行。别处一律炸弹能压 2。开源 XWeaponX7 是中间态：单张 2 只能被王或炸弹压。

### D22 ★【孤证】上海系「2 断牌改牌」
[简书随笔](https://www.jianshu.com/p/f2662cbbb281)：「2 可以**断牌和改牌**，例如前面人出了一张 7，后面人没有 8 能连，如果你手中有个 2，可以出来断牌后**改变出法，重新出单张、对子或三张以上连牌**」。这把 2 从「最大单张」改成了**牌型重置键**，是完全不同的机制。孤证，但机制独特到值得记一笔。

### D23 ★ 四川原生系的「对子压单张」
开源 LinneyRUNN✓：`对子接单牌：newMax === lastMax` —— **同点数的对子可以压单张**。别处无此说。

## 5. 歧义热点排序

按「民间吵架频度 × 平台实现分歧度 × 对 engine 的破坏力」综合排序。掼蛋的经验是规则歧义点＝bug 温床（那边是逢人配红心 2），这一节是本次调研对项目最有价值的部分。

| 排名 | 热点 | 为什么危险 |
|---|---|---|
| 1 | **摸牌模式**（D3） | 同一平台自家两页互相打脸；改变整局长度与胜负分布；AI 规划完全不同 |
| 2 | **要得起能不能过牌**（D1） | 两个不同的游戏；删掉一整层策略；JJ 与 QQ/91y 正面对撞 |
| 3 | **三张算不算炸**（D2） | 连带决定飞机牌型存不存在；5 张手牌下概率影响巨大 |
| 4 | **计分线性 vs 指数**（D12） | 结果差一个数量级，最容易当场翻脸 |
| 5 | **癞子牌型歧义消解**（D10） | 掼蛋逢人配同款坑的加强版；大一法则下不同解释**不等价** |
| 6 | **纯炸 vs 癞子炸谁大 / 炸弹比法**（D4、D5） | 四种做法互斥；软硬炸术语两地定义相反 |
| 7 | **春天/关门定义**（D13） | 四种并存，同站矛盾；结算时才暴露，属于「打完一局才发现规则不对」 |
| 8 | **连对最少几对**（D8） | 2 对 vs 3 对，边界穷举测试必须覆盖 |
| 9 | **王能不能既百搭又王炸 / 能不能当 2**（D9） | 两个流派互斥，不能同时开 |
| 10 | **报单罚则空白**（D18） | 全网无规定，纯靠临场约定 |
| 11 | **最后一张能不能留 2 / 3 是不是癞子**（D16、地域孤证） | 地域性强，实现前必须确认目标玩家群 |

**唯一全网口径一致、可以放心写死的**：大一法则、2 管任意单张、顺子最短 3 张且 2 不入顺、计分按剩余张数。

## 7. 给 engine 实现者的提醒

### 7.1 不先拍板就写不了代码的（阻塞项）

按 `RuleConfig` 开关来组织。以下三个是**结构性开关——会改变 AI 搜索空间**，必须先定，优先级最高：

```ts
// 结构性（必须先拍板，影响 AI 与状态机）
drawMode:        'all' | 'winnerOnly'      // D3
mustPlayIfAble:  boolean                    // D1
tripleIsBomb:    boolean                    // D2

// 牌型边界（影响 combos 穷举测试的全集）
minPairRunLength:   2 | 3                   // D8
maxRunRanks:        number | null           // D11
followRule:         'strictPlusOne' | 'equalOrPlusOne'   // D15

// 牌库与癞子（影响发牌与万能牌指派）
deck:        'single54' | 'double108' | 'jj58Wild' | 'plus1Wild55' | 'noJoker52'  // D6
wildKind:    'jokers' | 'dedicatedWild' | 'rotatingRank'   // D9/D20
wildCanBeTwo:  boolean                      // D9
jokerBombExists: boolean                    // D9（与纯赖子派互斥）

// 结算（可以后定，但要先留出接口）
bombMultiplierMode: 'linear' | 'exponential' | 'perType'   // D12
springDef: ...                              // D13
```

**最低限度必须现在拍板的是 D1 / D2 / D3 三条**，其余可以先按主流默认值实现、留开关。

### 7.2 术语陷阱（别用这些词做类型名）

- **「飞机」有三种互斥含义**：三张同点（四川民间）/ 连对（开源 tobughan 源码里就是这么命名的）/ 斗地主式三带牌。**engine 里禁用这个词**。
- **「软炸/硬炸」两地定义相反**（D5）。用正交字段 `{ bombSize: number, containsWild: boolean }` 表达，UI 文案再按配置映射。
- **「连对」在边锋叫「板凳」**，UI 文案要考虑。
- **「关门」有五种叫法**：关死/关牌/被关/关瞪瞪/春天。

### 7.3 与掼蛋的经验对应

- **癞子指派是同一个坑，而且更狠**：掼蛋逢人配的坑在「牌型识别 + AI 枚举要找最优指派」；干瞪眼因为有**大一法则**，`王+5+6` 解释成 `456` 还是 `567` **直接决定下家能不能接**，两种解释**不等价、不可事后推断**。协议层必须让出牌携带显式解释（阿拉游的弹窗、cdzwm 的「返回所有解释数组」都是这个思路）。必须有针对性穷举单测。
- **状态机比掼蛋多一个阶段**：掼蛋是 `play → play → ...`；干瞪眼是 `play(一圈) → 全员过 → draw(摸牌) → 新一轮领出`。**摸牌阶段是独立状态**，且摸牌顺序（从上轮最后出牌者开始逆时针）在牌堆见底时决定谁摸得到——pagat 特意强调了这点。别把 draw 塞进 play 的副作用里。
- **AI 不能从掼蛋移植**：大一法则下手牌价值函数完全不同——大牌不再天然值钱（你有 AKQ 也可能一手接不上），「手上有没有正好大一级的牌」比「牌大不大」重要得多。记牌器要记的是**各点数的剩余分布**而非大牌去向。

### 7.4 测试要点（写单测时先覆盖这些）

1. **大一法则的边界**：A 之后是不是 2？（不是——2 是特权单张，A 上面接不了普通牌）；顺子 QKA 之后能不能接？
2. **顺子/连对张数必须相等**：345 不能被 4567 接。
3. **2 的三条特权**互相独立：压单张、压对子、不入顺子/连对——每条各写测试。
4. **炸弹全序**：3 张 → 10 张 → 王炸，且要覆盖 D4 选定的比法（张数优先/点数优先/纯炸 vs 癞子炸）。
5. **癞子 0/1/2 张的最优指派**（掼蛋同款穷举打法）；两副牌时还要覆盖**同色双王不成王炸**。
6. **牌堆见底的摸牌顺序**：3 人 2 张牌时谁摸到。
7. **首攻不能过牌**、**只剩一张王的死局**。
8. **计分**：倍数连乘/线性/个人倍数三条路径各一组，含「春天」四种定义的分支。

### 7.5 建议的实现路线

先按**「主流民间版」**做一套默认配置——54 张、2–5 人、庄 6 闲 5、摸牌堆、大一法则、顺子 3 张起、2 不入顺、三张即炸、双王最大、王百搭不单出、可主动过牌、按剩牌张数 × 2^炸弹数计分——这套在三条证据链上重合度最高。然后把 D1–D3 做成房间选项。

**不要做的**：JJ 那套 58 张赖子 + 固定 3 人 + 有出必出，是竞技平台的商业改造版，不是民间通行玩法，别当默认。
