# 掼蛋联机（三期）设计

> 状态：设计待 owner 审核 · 日期 2026-06-21

## 问题

掼蛋一期（单局对 AI）、二期（整盘升级/进贡/打A）已完成并上线。三期让真人 4 人能在网上一起打整盘——这才是正常玩法；本地对 AI 降级为调试模式。

## 谁用 / 什么时候用

喜欢掼蛋的玩家（朋友、同事、网友），各自手机/电脑，建房邀请或随机匹配，凑齐 4 真人打一整盘。

## 定位调整（重要）

- **正常游戏 = 联机模式**：点「掼蛋」直接进联机（昵称 → 大厅/建房/匹配）。
- **本地对 AI = 调试模式**：藏到 `?debug` 入口，正常 UI 不露，供开发不起服务端就能调引擎/UI。
- 含义：正常玩依赖服务端；服务端挂了正常局玩不了，调试模式是开发兜底。

## 架构：权威服务端（方案 A）

掼蛋藏手牌（每人只看自己的牌），所以不能用 xiangqi 那种「纯哑中转 + 每端全量」（每端都能看到所有人的牌＝谁都能开挂）。改为**权威服务端**：

- 服务端跑掼蛋**引擎 + AI**、握全局牌局（4 手牌 + MatchState）。
- 每次状态变化推两类消息：
  - `state`（公开态）：发房内所有人 + 观众。
  - `hand`（私有手牌）：**只发给该座位本人**，别人的手牌永不下发。
- 客户端只发「我出这几张 / 不要 / 还这张」，服务端用 `isLegalPlay` 校验、`play/pass/applyTribute` 推进。
- 防作弊：别人的牌从不下发，客户端拿不到也伪造不了；互信范围只剩「同队线下串牌」这种软件层防不了的，不防（同 xiangqi）。

引擎是唯一真相，**服务端与客户端 import 同一份 `engine/`**，绝不各写一套规则。

## 产品边界（owner 已确认 2026-06-21）

| 维度 | 决定 |
|---|---|
| 开局 | 建房（6 位房号邀请）+ 随机匹配，都要 |
| 凑人 | **开局必须 4 真人**（不做 AI 补位开局）|
| 掉线 | 开打后掉线 **AI 接管继续**，不暂停；人回来**收回座位** |
| 范围 | **整盘**（跨局升级 + 进贡/还贡/抗贡 + 打A过A，二期 parity）|
| 观战 | 只看**公开信息**（出的牌/各家张数/谁赢），看不到手牌 |
| 座位/队友 | 建房**自己挑空位入座**（对门 0&2 / 1&3 自动一队）；匹配自动分座 |
| 身份 | 沿用 xiangqi：localStorage 昵称 + 服务端在线判重，无账号/落盘 |
| 局中加人 | **不允许**——AI 顶到整盘结束，结束后「再来一盘」才重开座位 |

## 房间 / 匹配 / 座位 生命周期

- **昵称**（连上先做）：`hello(昵称)` → 在线判重 → `hello-ok` / `nick-taken`。
- **建房**：`create` → 6 位房号、4 座、房主先挑座、`status=waiting`；牌友 `join(房号)` → `take-seat(i)` 挑空位；坐满 4 真人 → **房主点 `start`** → `playing` 发牌。
- **匹配**：`match` → FIFO 匹配池 → 够 4 → 自动建匿名房、自动填座 0~3、直接 `playing`。
- **大厅/观战**：`lobby` 订阅公开房列表；`spectate(房号)` → 观众集 → 只收公开态。
- **掉线 → AI 接管**：WS 断 → 标记该座 offline、保留昵称作重连凭据 → 该座轮到时 `choosePlay` 自动出，其余 3 人继续；广播 `peer-offline{seat}`。
- **重连 → 收回**：客户端把 房号+昵称+座位 存 sessionStorage → 断线自动 `rejoin(房号,昵称)` → 按房间座位昵称匹配，座位从 AI 收回真人 → **重发该座位完整态**（手牌 + 公开态 + 当前轮/进贡阶段）；广播 `peer-back{seat}`。
- **删房**：`waiting` 房房主离开 → 删房；`playing` 房只要 ≥1 真人在线就保留，**4 真人全掉线**才删房；整盘打完出结果 → 房主「再来一盘」（留座重开）或解散。

## 协议 + 数据流

**核心原则：手牌按座位私发，其余全公开。**

**客户端 → 服务端**

| 消息 | 含义 |
|---|---|
| `hello{nick}` / `rename{nick}` | 昵称登记/判重 |
| `create{isPrivate}` / `join{code}` / `take-seat{i}` / `start` | 建房/进房/挑座/房主开打 |
| `match` / `lobby` / `spectate{code}` | 匹配/大厅/观战 |
| `play{cardIds}` / `pass` | 出牌/不要（只发牌 id，服务端校验归属 + isLegalPlay）|
| `tribute-return{cardId}` | 还贡：选一张 ≤10 的牌还给出贡那家 |
| `rejoin{code,nick}` | 断线重连 |

**服务端 → 客户端**

`hello-ok` / `nick-taken`、`created{code}`、`room{seats,status}`（座位/谁坐哪/在线 or AI）、`started`、`state{公开态}`、`hand{cards}`（私有，只发本人）、`need-tribute{可选牌}`、`result{名次/比分/级别}`、`peer-offline{seat}` / `peer-back{seat}`、`error{msg}`。

**公开态 `state` 字段**：各家张数、桌面各家上一手（出的牌 / 不要）、当前轮到谁、级别（我方/对方打几）、进贡阶段、本局名次、整盘比分、谁在线/谁 AI。

**进贡 / 还贡（整盘最特殊的一段）**

- 一局结束 → 服务端 `planTribute` 算谁进贡给谁、是否抗贡。
- **进贡牌 = 一手中除逢人配（红心级牌）外最大的一张，大小王照样列入**（大王＞小王＞级牌＞A…；只有逢人配不作进贡牌，Q3）。单下、双下都一样。
- **进贡**自动（系统按上述规则抽末游[/三游]的进贡牌），服务端执行 + 广播。
- **双贡**（双下：头游+二游同队）：末游、三游各拿各自的进贡牌，**头游收点数较大的、二游收较小的**（与谁出无关；引擎 `planTribute` 已实现）。服务端给头游、二游两个真人座位各发 `need-tribute`，各自还给出贡那家。
- **还贡**交互：服务端给收贡的真人座位发 `need-tribute{可选 ≤10 牌}` → 客户端弹层选 → `tribute-return{cardId}` → 校验 + 执行。
  - 收贡座位是 AI → 服务端 `autoReturn` 自动还。
  - 超时（~30s 未选）→ `autoReturn` 兜底，防卡全场。

## 客户端驱动层改造

- 抽 `GameDriver` 接口：view 只依赖 driver 给的 **view-model**（我的手牌、各家张数、桌面各家上一手、当前轮、级别/进贡阶段、名次、比分）+ `onUpdate` 回调，调 `driver.play(ids) / pass() / returnTribute(id)`。
- `LocalDriver`（调试）：现在的本地引擎 + AI 逻辑搬进来，同步推进。
- `OnlineDriver`（正常）：操作发 WS；收服务端 `state`/`hand` → 拼 view-model → onUpdate；自身无规则/AI。
- view-model 是关键缝：服务端公开态 + 私有手牌拼出跟本地一样的 view-model → **牌面渲染零改动**（一/二期打磨的透明/层叠/不要按钮等全继承）。
- 入口：点掼蛋 → OnlineDriver（昵称/大厅/建房/匹配前置 UI）；`?debug` → LocalDriver（直接发牌对 AI）。

## 测试策略

- **服务端纯逻辑单测**（.mjs，照 xiangqi rooms.test）：4 座房间（建/挑座/满员开打/匹配凑4/观战/重连/掉线AI接管/删房）。
- **match-driver 单测**：喂操作序列，验手牌只发本人、公开态正确、非法操作拒、掉线 AI 续、进贡/还贡含双贡。
- **引擎单测**：全继承（已 238 测试全绿，含 1000 局 + 50 整盘 fuzz）。
- **多页 E2E**（Playwright 多 context）：4 端建房/挑座/开打/互看/一人掉线AI接管/重连收回/观战只公开/打完整盘——联机真路径冒烟。

## 部署（细节进 DEPLOY.md，公网信息不入库）

- 复用 xiangqi 套路：服务端 systemd 常驻（自选端口避开 80/8080）、Let's Encrypt DNS-01（CF）、`CERT_DIR` 有证书则 https/wss。
- desk-games 现纯静态托管在 18080 → 联机换成「server.mjs 同时托管 dist + /ws」。掼蛋联机服务端是 desk-games 自己的，与象棋服务端各自独立。
- HTML 响应带 `Cache-Control: no-store`（同 xiangqi，防 iOS Safari 缓存旧代码）。

## 非目标（一期联机不做）

- 账号 / 登录 / 落盘 / 战绩（无账号、纯内存）。
- 匹配段位 / 筛选（只 FIFO 凑 4 人）。
- 局中加人 / 换人（AI 顶到整盘结束）。
- 观战上帝视角（只公开态）。
- 聊天 / 表情 / 语音。
- 防作弊到「同队线下串牌」级别（互信）。

## 验收标准

- 4 个真人能建房/挑座/开打，或随机匹配凑 4 人开打。
- 全程每人只看到自己的手牌（抓包验证别人手牌不下发）。
- 一人掉线，牌局不中断、AI 接管；该人重连后收回座位、看到正确当前态。
- 整盘流程正确：跨局升级、进贡/还贡（含双贡头游拿大牌）、抗贡、打A过A，与二期一致。
- 观战只看公开信息。
- 服务端 + match-driver 单测全绿；多页 E2E 打完一整盘通过。
