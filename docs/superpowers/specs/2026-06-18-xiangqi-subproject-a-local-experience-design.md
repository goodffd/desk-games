# 象棋 v3 路线图 + 子项目 A（本地体验增强）设计

日期：2026-06-18
状态：设计已锁定，待写实现计划
范围：本文档涵盖 v3 整体路线图，以及第一个子项目 A 的完整设计。B/C 各自另立 spec。

---

## 1. 背景与缘起

当前 `xiangqi-game` 已完成 v1（完整规则双人对弈）+ v2（长将/长捉循环裁决 + α-β AI）。
本轮 v3 把 SPEC.md 原「非目标」里的三组功能提升为目标：

1. 联网/在线对战、存档读档、棋谱导出（PGN/ICCS）
2. 计时器、音效、主题皮肤
3. 开局库、残局库、超出长将/长捉的整本竞赛规则细分裁决

这三组是**体量与风险天差地别的独立子系统**，不能塞进一个 spec。决策：拆分，逐个走 spec→计划→实现。

## 2. 拆分与排期（A → B → C）

排序依据是**真实依赖关系**，不是难度：

- **A 会建出一层「序列化地基」**：局面 FEN、中文+ICCS 着法记谱、PGN 整盘容器。
- B 的开局库/残局库要「加载局面/着法」→ 复用 A 的反序列化。
- C 的联网对战要「两端同步局面、传输着法」→ 复用 A 的着法编码。

故 A 是 B、C 的前置，必须先做，否则三处各写一套序列化逻辑、最终对不齐。

| 子项目 | 内容 | 架构影响 | 风险/体量 |
|---|---|---|---|
| **A 本地体验增强** | 存档读档 · 棋谱导出(PGN/ICCS) · 计时器 · 音效 · 主题皮肤 | 纯前端，守单文件双击 | 低 / 中 |
| **B 规则与棋库深化** | 整本竞赛规则细分裁决(B1) · 残局库 · 开局库 | 纯前端，吃 engine+穷举单测红利 | 规则裁决高 / 库类中 |
| **C 联网在线对战** | 在线对战 | **需后端，破坏纯前端架构** | 高 / 大，持续运维 |

约定细分：
- **A 内部先做 A1 序列化地基**（B/C 前置），再做 A4 主题、A2 棋钟、A3 音效（后三项相互独立）。
- **B1 整本竞赛规则细分裁决单独立 spec**：难度与测试量级远超 B2/B3，混在一起会被拖垮；必须以《中国象棋竞赛规则》等权威文本逐条做依据。
- **C 启动前单独拍板架构**：自建 WebSocket 服务器（要部署运维）vs WebRTC/P2P 直连（省服务器、有 NAT 穿透坑）vs 第三方实时服务。本轮不细化。

## 3. 不变的铁律（贯穿三个子项目）

- `src/engine/` 绝不 import DOM/Canvas，纯函数可被 vitest 穷举单测——这是「无 bug」的核心杠杆。
- engine 是规则与数据的唯一真相，UI 只调用其公开接口、不复制逻辑。
- 零资源：不引图片/音频文件，棋盘棋子程序化绘制、音效合成生成。
- 单文件：`npm run build` 产出可双击的单 HTML，`file://` 下正常运行。

## 4. 子项目 A：设计详情

### 4.1 模块划分

新增 6 个、改动 3 个，沿用现有 `engine/`（纯逻辑）/`ui/`（展示+接线）分层。

| 文件 | 增/改 | 职责 | 依赖 |
|---|---|---|---|
| `engine/fen.ts` | 新 | 局面 `Board`/轮走方 ↔ 象棋 FEN 字符串 | types |
| `engine/notation.ts` | 新 | `Move` ↔ 中文记谱 / ICCS 坐标，双向 | types, board |
| `engine/pgn.ts` | 新 | 整盘 ↔ PGN 式文本（tags + 着法表 + 起始 FEN） | notation, fen |
| `engine/clock.ts` | 新 | 棋钟纯状态机（包干/读秒、tick、到点判负） | types |
| `ui/sound.ts` | 新 | Web Audio 合成四类音效 + 静音开关 | — |
| `ui/themes.ts` | 新 | 四套 Theme 调色板 + 棋子风格枚举 | — |
| `ui/render.ts` | 改 | 颜色全从 Theme 读、按棋子风格分发绘制 | themes |
| `ui/controller.ts` · `ui/main.ts` | 改 | 接线：存读档/导入导出、棋钟、音效、主题切换+记忆 | 上述全部 |
| `index.html` | 改 | 新增按钮/下拉：存档·读档·导出·导入·棋钟设置·静音·主题 | — |

### 4.2 数据格式

**FEN（象棋通行约定）**：10 行（黑在上、红在下）以 `/` 分隔，从行内左到右；大写=红、小写=黑；
字母 `K`将帅 `A`士仕 `B`象相 `N`马 `R`车 `C`炮 `P`兵卒；数字表连续空位；尾部附轮走方（`w`/`b` 或 `r`/`b`，实现时定一种并注释）。
选标准格式是为了 B 的开局/残局库能直接导入外部数据。

**着法记谱（双轨）**：内部 `Move` 对象不变；`notation.ts` 负责渲染/解析两种字符串：
- **中文记谱**：`<子><纵线><进/退/平><目标>`。红方纵线用汉字 `九八七六五四三二一`（自右向左数），黑方用 `1-9`（自左向右数）；
  同一纵线有两个同类子时用「前/后」消歧（如「前马」「后马」）；同线多兵卒用「一二三四五」从前到后消歧。**这是最易错点，重点测试。**
- **ICCS 坐标**：纵线 `A-I`（红视角左到右）、横线 `0-9`，着法记为 `起点-终点`（如 `H2-E2`）。

**存档**：
- 导出 `.pgn` 文本：元信息 tags（`[Event][Date][Red][Black][Result][TimeControl]`）+ 着法表（中文与 ICCS 并存）+ 起始 FEN（默认开局可省）。
- localStorage JSON 快照：当前 `GameState` + 棋钟剩余 + 选用主题 + 静音状态，用于「继续上局」。

### 4.3 主题架构（唯一有取舍处）

现状 `render.ts` 颜色写死、棋子是「立体骨牙阴刻」单一画法。四套主题不是单纯换色：
- **A 朱砂水墨、B 原木棋枰**：同一「牙盘阴刻字」画法，仅换调色板。
- **C 夜间墨玉**：暗盘 + 发光阴刻字。
- **D 素雅纸枰**：实色双拼盘 + 白字（扁平）。

设计：`Theme = { palette, pieceStyle }`，其中 `pieceStyle ∈ {ivory, luminous, solid}`；`render.ts` 按 `pieceStyle` 分发到不同绘制分支。
`render` 仍是纯展示，签名扩展为输入 `Theme + GameState`。四套调色板（沿用 brainstorm 对比页定值）：

| 主题 | 棋盘底 | 线 | 红 | 黑 | 高亮 | pieceStyle |
|---|---|---|---|---|---|---|
| A 朱砂水墨(默认) | 暖宣纸渐变 | `#3a332a` | 朱砂 `#b5362a` | 徽墨 `#2a2018` | 青玉 `#3f6b5e` | ivory |
| B 原木棋枰 | 木纹 | `#6b431f` | 红 `#a83224` | 绿 `#1f6b46` | `#2f6b46` | ivory |
| C 夜间墨玉 | 深岩渐变 | `rgba(150,180,195,.42)` | 暖红 `#ff7d6a` | 青 `#74dcc6` | `#6fe0cc` | luminous |
| D 素雅纸枰 | 米白 | `#c2b596` | 实红 `#c0392b`+白字 | 炭 `#2f2f33`+白字 | `#3f8f7d` | solid |

主题选择存 localStorage 记忆；切主题不得改变 engine 任何判定。

### 4.4 棋钟

`engine/clock.ts` 纯状态机：
- 模式：`banker`（包干，每方一个总时限）/ `byo-yomi`（读秒，基本时间用尽后每步固定秒数）。
- 状态：红黑各自剩余、当前读秒标记、走方。
- 接口：`tick(state, elapsedMs)`、`onMove(state)`（切走方/结算读秒）、`isFlag(state)`（是否超时）。
- 参数可配：总时长、读秒秒数。

UI 用 `setInterval`/`requestAnimationFrame` 驱动 `tick`；`isFlag` 为真 → 接入现有终局流程判负（复用胜负提示 + 音效）。
悔棋：时间随局面快照一并回退到上一手（快照里带棋钟剩余）。

### 4.5 音效

`ui/sound.ts` 用 Web Audio `OscillatorNode + GainNode` 包络合成四类短音：
落子（清脆木击）、吃子（略重）、将军（警示双音）、胜负（终局和弦）。
首次用户交互后 `AudioContext.resume()`（合规浏览器自动播放策略）。静音开关存 localStorage。零音频文件、`file://` 可用。

### 4.6 接线（controller/main/index.html）

- index.html 增控件：存档/读档、导出/导入（`.pgn`）、棋钟设置（模式+参数）、静音切换、主题下拉。
- 文件导出用 `<a download>` + Blob；导入用 `<input type=file>` 读文本 → `pgn.ts` 解析 → 重放。`file://` 下两者均可用。
- controller 在每步落子后：触发音效、更新棋钟、写 localStorage 快照。

## 5. 测试策略（守「无 bug」硬指标）

引擎层穷举单测（纯函数，本项目核心杠杆）：
- **notation**：七种子中文/ICCS 双向往返 `parse(format(m)) === m`；同线双子「前/后」消歧；同线多兵卒「一二三四五」消歧；ICCS 边界。
- **fen**：开局 FEN 往返；多个中局局面往返；非法 FEN（错列数/非法字符/缺王）被拒。
- **pgn**：整盘对局导出→导入→重放，逐手局面一致；元信息 tags 正确读写。
- **clock**：包干耗尽 `isFlag` 真并判负；基本时间用尽切读秒；悔棋回退时间快照。

UI 层不写脆弱像素测试，但：
- 保证「切主题不改变 engine 行为」（render 仍只读 GameState，不回写）。
- 浏览器真机验收（见下）。

验收（真路径实证）：
- 四套主题一键切换显示正确、判定不变。
- 存档后关页面再开能续局；导出 `.pgn` 能被重新导入还原。
- 棋钟到点判负并提示；四类音效触发且可静音。
- `npm test` + `npm run typecheck` 全绿；`npm run build` 单文件 `file://` 正常。

## 6. 实现顺序

1. **A1 序列化地基**：`fen.ts` → `notation.ts` → `pgn.ts` → 存读档接线（B/C 前置，先做）
2. **A4 主题皮肤**：`themes.ts` + `render.ts` 重构 + 切换接线
3. **A2 棋钟**：`clock.ts` + UI 驱动 + 到点判负
4. **A3 音效**：`sound.ts` + 触发接线

每步先加/改单测、红→绿，再接 UI；禁止注释失败用例绕过。

## 7. 明确不做（子项目 A 范围内）

- 加秒(Fischer)、中式三段等小众计时制 → 只做包干 + 读秒。
- WXF 记法 → 只做中文 + ICCS。
- 账号/排行榜/原生 App/强引擎 → 全 v3 范围外。
