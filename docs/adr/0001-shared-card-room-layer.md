# 抽出牌类公共房间层，象棋不并入

掼蛋自带约 1200 行房间 / 协议 / 会话代码（`server/guandan-rooms.mjs` + `src/games/guandan/online/`），象棋另有一套独立的 158 行 `server/rooms.mjs`，两者互不共用。加入干瞪眼时若照旧，就是把大厅 / 昵称 / 房间 / 座位 / 重连这套东西**抄第三遍**。因此把它抽成牌类公共房间层，由掼蛋与干瞪眼共用。

## Considered Options

- **各游戏各抄一套（现状延续）**：零重构风险，但第三份重复代码之后，任何一个联机 bug 都要修三处。
- **抽公共层，三个游戏全并入**：最彻底，但象棋是 2 人棋类——没有牌、没有 AI 补空座、胜负判定与重连语义都不同，硬并进来只会让公共层长出「只有象棋会走」的分支，抽象反而被污染。
- **抽公共层，只覆盖牌类（选定）**：掼蛋与干瞪眼共用；象棋那 158 行原样不动。

## Consequences

- 公共层的边界是**座位与转发**：连接、昵称、大厅列表、建房 / 加入 / 观战、座位认领、房主开打、掉线重连、AI 补空座位。它**不解析任何对局消息**，只按座位转发不透明 payload，游戏侧只需提供一个 `MatchRunner`。因此公共层里不会出现任何一个游戏的术语。
- 必须参数化 `minSeats / maxSeats`：掼蛋固定 4 人，干瞪眼 2–5 人且由房主建房时选定目标座位数。
- **掼蛋要能原样跑在这层上，才算抽对了。** 掼蛋已上线且稳定，切换后必须跑通现有 `guandan-rooms.test.ts` / `online-*.test.ts` 全绿 + 真机多 context 冒烟（建房 / 加入 / 对弈 / 重连），确认零回归。
- 象棋被显式排除。日后若真要并入，需要新开 ADR 推翻本条——不要因为「看起来也是房间」就顺手合并。
