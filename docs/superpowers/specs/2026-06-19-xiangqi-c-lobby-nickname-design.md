# 子项目 C 增补：房间大厅 + 昵称 + 观战 + 私密房 设计

日期：2026-06-19
状态：设计待 owner 过目（已含多轮澄清结论）
范围：把联机从「纯配对码」升级为「房间大厅 + 昵称 + 观战」，保留私密房（仅凭码加入）。基于已上线的 C（自建 WS 哑中转，已部署到 owner 自托管服务器）。

## 问题
现状：一方建房拿 6 位码、口头报给另一方输码配对。owner 的私有服务器场景（URL 只发家人朋友、进来的人本就可信），配对码是多余摩擦。改为：谁建房就显示在大厅，点击加入；并要看得出"谁是谁"，且能观战正在进行的对局。

## 决策（owner 多轮拍板）
1. **大厅替代纯配对码**：公开房显示在大厅，点击加入/观战。
2. **昵称 = 本机记名 + 服务器在线判重**（不做账号/不落盘/不碰持久化红线）：
   - 用户群是**不同的人、各用各的设备**（电脑/手机/pad）。每个人在自己设备上设一次名，localStorage 记住、下次自动填——本机存储恰好"每台设备记住用它的人"。
   - 服务器对**当前在线**昵称判重：撞名拒绝、提示改名。离线后名字不被永久占用（互信场景可接受，不做永久预留）。
   - 共享设备（如家用 iPad 多人轮流）：名字框可编辑，谁用谁改一下（自动填上一个人）。
   - **明确不做**：注册/登录/密码/PIN、服务器落盘存名单、跨设备同一身份、排行榜/积分/社交资料。
3. **观战**：大厅也列"对局中"的公开房，可作为观战者只看不动。
4. **私密房开关**：建房可勾「私密」→ 不进大厅、仅凭房间码加入、不可观战（仅玩家）。

## 组件

### 1. 昵称（前端 persist + UI + 在线判重）
- `persist.ts` 新增 `nick`（localStorage）。无值时生成友好默认名（如「棋友」+3 位随机数）并写回。
- **进联机第一步是"输名字"关**：显示名字输入框（localStorage 有则预填上次的名、无则填默认名）+「进入」按钮。用户确认后发 `{t:'hello', nick}` 判重：占用回 `{t:'nick-taken'}` → **停在输名字这步**、提示"该昵称已被占用，换一个"、聚焦输入框，改名前不进大厅；可用回 `{t:'hello-ok'}` → 写回 localStorage 并进入大厅。localStorage 只负责预填方便回头用，判重始终走这道确认。
- 联机面板顶部「我：<昵称输入> [改名]」。改名发 `{t:'rename', nick}`，服务器同样判重。
- 后续 create/join/spectate 用已登记的昵称。

### 2. 服务器大厅 + 观战（`rooms.mjs` + `server.mjs`）
- **在线昵称集**：`Map<conn, nick>` + `Set<nick>`，连接 hello 时登记、断开时释放、rename 时更新；判重据此。
- **房间模型扩展**：`{ code, host(创建者昵称), isPrivate, players:[redConn|null, blackConn|null], playerNicks:[red,black], spectators:Set<conn>, status:'waiting'|'playing' }`。
- **大厅订阅集**：一组"正在看大厅"的 conn。
- **大厅快照** = 所有**非私密**房，每项 `{code, host, status, players:[redNick,blackNick]|null, spectators:数量}`；status=waiting（1 人等待）或 playing（2 人对局）。快照变化（建房/满员/房空/观战人数变）→ 广播给订阅者。
- **观战中途同步（复用 A 的序列化，服务器仍不懂棋规）**：观战者发 `{t:'spectate', code}` → 服务器把其加入 `spectators`、回 `{t:'spectating', host, players}`，并向房内一名玩家发 `{t:'need-sync'}`；该玩家回 `{t:'sync', pgn}`（用 A 的 `gameToPgn` 序列化当前整盘）→ 服务器转发给该房**待同步的观战者**；观战端用 `pgnToGame` 载入瞬间追到当前局面（无动画），之后实时着法正常处理。
- **转发**：房内 `move/resign/draw-*/undo-*` 转发给**对手 + 全部观战者**（原 1→1 改 1→N）。
- **断开**：玩家断 → 对手收 `{t:'peer-left'}`、房间清理（含通知观战者 `{t:'room-closed'}`）；观战者断 → 仅从 spectators 移除（玩家无感）、更新大厅观战数。
- 仍**无数据库、无落盘**（在线昵称集 / 房间表 / 订阅集全内存）。

### 3. 消息协议（在现有基础上增补）
身份/大厅：
- `{t:'hello', nick}` → `{t:'hello-ok'}` | `{t:'nick-taken'}`
- `{t:'rename', nick}` → `{t:'rename-ok'}` | `{t:'nick-taken'}`
- `{t:'lobby'}`（订阅）→ `{t:'lobby', rooms:[…]}`；服务器在快照变化时持续推 `{t:'lobby', rooms:[…]}`
- `{t:'create', isPrivate}` → `{t:'created', code, isPrivate}`（host=已登记昵称；公开房进大厅并广播）
- `{t:'join', code}` → 双方 `{t:'paired', color, you, opponent}`（先到 red 后到 black，附双方昵称）；满员/坏码 `{t:'error', msg}`；房转 playing → 广播大厅
- `{t:'spectate', code}` → `{t:'spectating', host, players}` + 触发 need-sync/sync 同步流；坏码/私密 `{t:'error', msg}`
- `{t:'need-sync'}`（服务器→玩家）/ `{t:'sync', pgn}`（玩家→服务器→观战者）
- `{t:'peer-left'}` / `{t:'room-closed'}`
对局转发（房内复用现有，1→N）：`move/resign/draw-offer|accept|decline / undo-request|accept|decline`。

### 4. 前端联机 UI（`online.ts` + `main.ts` + `index.html`）
- 进联机：**先到"输名字"步**（输入框 localStorage 预填/默认名 +「进入」按钮）→ 确认发 hello 判重；撞名就地提示改名、不放行；通过后连大厅订阅 → 按 `lobby` 渲染列表。进入大厅后顶部「我：<昵称> [改名]」可再改（改名走 rename 判重）。file://（无 host）→ 禁用并提示，同现状。
- 面板：
  - 顶部「我：<昵称> [改名]」（判重失败就地提示改名）
  - 「创建房间」按钮 +「私密」勾选框
  - **房间列表**（非私密，自动刷新）：每行显示状态——
    - 等待中：「<host>的房间 · 等待中  [加入]」
    - 对局中：「<红> vs <黑> · 观战 N  [观战]」
    - 空列表提示"暂无房间，点上面创建一个"
  - 「或 输房间码加入：<输入> [加入]」（私密房 / 直接邀请用）
- 创建后：公开房"等待对手加入…（已在大厅）"+ 可选展示码；私密房突出展示码供分享。
- 加入：列表「加入」发 `join(该房 code)`；配对成功显示对方昵称、进对局。
- 观战：列表「观战」发 `spectate(code)` → 收 spectating + sync → 载入当前局面进入**只读观战视图**（棋盘不接受点击，顶部标「观战中 · <红> vs <黑>」，无在局控件/认输求和）；实时着法动画更新；房关闭提示并退回大厅。
- 对局界面：显示双方昵称（红·<昵称> / 黑·<昵称>）。本地双人/人机无昵称，维持现状文案。

## 非目标（本次增补内）
- 不做账号/注册/登录/密码/PIN、不做服务器落盘存名单、不做跨设备同一身份。
- 不做排行榜/积分/社交资料、不做永久昵称预留。
- 不做断线重连（断线即结束/退出观战）。
- 不做聊天、房间密码、大厅分页/搜索（家人朋友规模，全列）。
- 不防作弊（互信，服务器不校验棋规；观战同步靠玩家上报 PGN）。

## 已知限制
- 大厅公开房任何持有 URL 的人可加入/观战；要私密用「私密房」。
- 昵称仅在线判重、不跨设备、离线不预留；共享设备需手动改名。
- 观战中途同步依赖一名玩家在线上报 PGN；若该玩家恰好此刻断开，观战同步失败 → 提示重试。
- 创建者长期挂等待 → 房间留在大厅，靠 WS 断开清理（不加空闲超时）。
- 列表并发抢加入：先到成功，后到 error + 大厅刷新。

## 验收标准
- **server 单测**（Node，多个内存 ws 客户端）：
  - hello 判重：占用回 nick-taken、释放后可用；rename 判重；
  - 订阅 lobby 返回当前非私密房快照（含 waiting/playing 状态与昵称）；
  - 建公开房 → 订阅者收到含该房(host)的 lobby 广播；私密房不出现在 lobby；
  - 从列表 join → 双方 paired 携带正确 you/opponent 与颜色；房转 playing 并广播；
  - 满员/坏码 → error；
  - **观战**：spectate 公开对局房 → 收 spectating + 收到经服务器转发的玩家 sync(pgn)；此后玩家 move 同时送达对手与观战者；观战者断开仅减观战数不影响玩家；
  - 私密房不可 spectate（error）；
  - 等待中公开房创建者断开 → 房清理 + lobby 广播；对局中玩家断开 → 对手 peer-left + 观战者 room-closed。
- 前端 online 编解码/派发可测部分单测。
- **端到端冒烟**（本机起 server + 3 个 Playwright 页面）：
  - A 设昵称建公开房 → B 大厅见「A的房间·等待中」→ B 加入 → 双方 paired、互见昵称、走子同步；
  - C 打开联机 → 见该房变「A vs B·观战」→ C 点观战 → 瞬间同步到当前局面 → 后续 A/B 走子 C 实时跟进（只读）；
  - 撞名：第二个页面用相同昵称 → 收 nick-taken、改名后进；
  - A 建私密房 → C 大厅看不到 → 仅凭码加入、不可观战；
  - 认输/求和/悔棋各往返一次、双方+观战者状态一致；零 console 错误。
- `npm test` + `npm run typecheck` + `npm run build` 单文件全绿；既有功能（本地双人/人机/棋钟/音效/主题/开局残局/存档/嵌入字体）不回归。
- 大厅 + 观战 + 对局昵称界面截图 owner 点头。
- 重新部署到服务器并真路径实测（公网 3 浏览器：大厅配对 + 观战）。

## 实现顺序
1. `persist.ts` 昵称存取 + 默认名。
2. `rooms.mjs`：在线昵称集 + 判重；大厅（订阅/快照/广播）；create(isPrivate)/join/spectate；玩家与观战者 1→N 转发；need-sync/sync 同步流；断开清理。+ Node 单测。
3. `server.mjs` 接线（hello/lobby/spectate、断开广播）。
4. `online.ts` 客户端（hello/判重/rename、lobby 订阅/列表、create/join/spectate、paired/spectating/sync 派发）。
5. 前端联机 UI 重构（昵称栏 / 创建+私密 / 房间列表含状态与加入·观战 / 输码入口 / 只读观战视图 / 对局昵称）+ 派发。
6. 3 页端到端冒烟（含观战、撞名、私密）+ 截图。
7. 重新构建部署 + 公网实测。
