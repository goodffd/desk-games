# Context Map

desk-games 是游戏厅，每个游戏是一套**互不相通的领域语言**。同一个词在不同游戏里含义不同——「顺子」在掼蛋是 5 张、在干瞪眼是 3 张；「炸弹」在掼蛋 ≥4 张、在干瞪眼 ≥3 张。因此这里不做统一术语表，按上下文分册。

## Contexts

- [干瞪眼](./src/games/gandengyan/CONTEXT.md) — 大一法则出牌类，1 副 54 张，2–5 人
- 掼蛋（`src/games/guandan/`）— 升级类 2v2，2 副 108 张。术语表尚未成文，以 `SPEC.md`《掼蛋规则》节为准
- 中国象棋（`src/games/xiangqi/`）— 2 人棋类。术语表尚未成文，以 `SPEC.md`《中国象棋》节为准
- 牌类公共房间层（规划中，见 [ADR-0001](./docs/adr/0001-shared-card-room-layer.md)）— 连接 / 昵称 / 大厅 / 房间 / 座位 / 重连

未成文的上下文不是遗漏，是**惰性创建**：等实际有术语要澄清时再落笔。

## Relationships

- **游戏厅壳 → 各游戏**：只经 `GameModule { id, name, desc, mount(root) }` 接入，游戏之间零依赖、零共享术语。
- **牌类公共房间层 → 掼蛋 / 干瞪眼**：公共层只认「座位」和「不透明 payload」，**不解析任何对局消息**，因此不引入任何游戏术语。
- **中国象棋**：不接入牌类公共房间层，保留自有的 `server/rooms.mjs`（见 ADR-0001 的显式排除理由）。
