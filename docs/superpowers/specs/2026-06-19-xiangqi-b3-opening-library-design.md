# 子项目 B3：开局库 设计

日期：2026-06-19
状态：设计已锁定，待写实现计划
范围：B3 开局库（B 的第一块）。B1 整本竞赛规则细分裁决、B2 残局库各自另立 spec。

## 问题
让用户既能**离线打谱学开局**，又能**对局中看书上谱着、知道何时出谱**。

## 谁用 / 何时用
封福东及家人朋友：想系统学几套主流开局时进"开局库"逐手打谱；随手对弈想练开局时开"开局提示"，看当前局面书上怎么走、走偏了有提示。

## 决策（brainstorm 已定）
- **两者都要**：独立浏览学习库 + 对局中实时谱着提示/出谱检测。
- **数据规模**：10+ 套主流开局，含多变着；手写嵌入代码，守零依赖/单文件。
- **浏览布局**：单列居中（顶部开局下拉、棋盘居中、谱着横排在下 + 步进），与现有对弈界面同构。
- **提示呈现**：文字行（开局名徽标 + 中文续着）+ 棋盘金色书签点高亮续着落点；**开局提示可开关、默认关**。出谱时徽标转「已出谱」、高亮消失。

## 架构

### 数据层 `src/engine/openings.ts`（纯逻辑，浏览/提示共用，可单测）
- `OPENINGS: Opening[]`：命名开局，**中文记谱手写成着法树**（主线 + 变着）。选中文记谱因其最自然好校对，且每步经 `chineseToMove`（notation.ts）解析 + 引擎重放校验合法性——写错/非法即失败。
  ```ts
  interface BookNode { zh: string; comment?: string; children: BookNode[]; }
  interface Opening { id: string; name: string; roots: BookNode[]; } // roots 通常单根（红首着）
  ```
- `buildBookIndex(): BookIndex`：DFS 重放每条谱线，从初始局面起逐手 `chineseToMove`→`applyMove`，在每个节点用 `toFen(board, turn)`（fen.ts）作键，记录 `{ 续着: {move: Move, zh: string}[], openings: string[] }`。天然处理异途同归（同 FEN 合并续着）。重放遇非法着法**抛错**（被测试捕获）。
- `lookupBook(index, board, turn): { moves: {move, zh}[]; openings: string[] } | null`：命中返回续着+开局名，未命中返回 null（出谱）。

### 界面一·开局库浏览（单列居中，独立模式）
- 现有模式按钮（双人/人机）扩展加「开局库」模式。进入后：
  - 顶部开局下拉（`OPENINGS` 列表）。
  - 棋盘显示当前打谱局面（由所选谱树驱动，非实时对弈）。
  - 下方中文谱着横排列表，当前手高亮；上一步/下一步步进；分支节点显示"变着"选择。
- 复用 `render`（棋盘）、`notation`（中文，数据里已是中文）、`board`/`applyMove`（推进局面）。退出回对弈模式。

### 界面二·对局中谱着提示（对弈模式，默认关开关）
- 对弈控件加「开局提示」开关（默认关，存 localStorage）。
- 开启后开局即、每步落子后 `lookupBook(index, controller.board, controller.turn)`：
  - 命中：棋盘把续着落点画**金色书签点**（区别青玉选中/着法点）；文字行显示开局名徽标 + 中文续着。
  - 未命中：文字行徽标转「已出谱」，无金色高亮。
- `render.ts` 末位加可选 `bookHints: Square[]`，画金色书签点（描环 + 实心点）。

## 非目标（B3 内）
- 出谱后的"推荐改正"或胜率评估（只提示在不在书上）。
- 用户自定义/导入开局（只读内置库）。
- 残局库（B2）、规则裁决（B1）。

## 验收标准
- **openings 单测**：`buildBookIndex` 对全书重放无异常（每条谱线全合法——自动抓手写错）；`lookupBook` 在初始局面/中炮开局若干已知局面返回正确续着与开局名；明显出谱局面返回 null；异途同归（两条谱线殊途同归到同 FEN）续着正确合并。
- **浏览模式**：选开局→逐手前进/后退局面正确；分支处可切变着；中文谱着列表与局面同步。
- **对局提示**：开关默认关、可开、状态持久化；开启后命中显金色书签点+开局名+中文续着，出谱显「已出谱」；切换提示不影响 engine 判定。
- `npm test` + `npm run typecheck` 全绿；`npm run build` 单文件。
- 真机冒烟：浏览步进、提示开关/命中/出谱；金色书签点观感截图 owner 点头。

## 实现顺序
1. `engine/openings.ts`：类型 + 数据（先少量、跑通 index/lookup）+ `buildBookIndex` + `lookupBook` + 测试；再补足 10+ 套。
2. `render.ts`：加 `bookHints` 金色书签点层。
3. 对局提示接线（开关 + 查书 + 文字行/徽标 + 出谱）。
4. 开局库浏览模式（下拉 + 步进 + 变着 + 谱着列表）。
