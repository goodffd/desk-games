# 干瞪眼 · 开发路线（四阶段）

> 规格真相：`SPEC.md`《干瞪眼》节。术语：`src/games/gandengyan/CONTEXT.md`。架构决策：`docs/adr/0001`、`docs/adr/0002`。
> 本文只定阶段边界与验收，每阶段动工前另写可执行 plan（task-by-task）。

**Goal:** 把干瞪眼做成 desk-games 第三个内置模块——2–5 人联机，单机 = 建房 + AI 补空座，规则零错。

**Global Constraints**
- `engine/` 是规则唯一真相，UI / AI / 服务端一律不复制判定逻辑，也绝不 import DOM。
- 规则与 AI **只在服务端**跑；客户端不写会被扔掉的临时单机驱动。
- 出牌一律携带**显式王指派**，engine 只校验不推断（ADR-0002）。
- 术语禁用「飞机」「软炸 / 硬炸」，炸弹一律用「张数 + 是否含王」两个正交属性表达。
- 每阶段验收：`npm test` 全绿 + `npm run typecheck`，改规则先加 / 改单测、红 → 绿。

---

## 阶段 0：工程前提（不做完，后面的测试活不下来）

- **测试分组**：`npm test` 现在 8 分 50 秒，掼蛋单局 fuzz 一个文件占 528 秒。干瞪眼再加一套 fuzz + 座位数维度就直奔 15 分钟，到那时没人会在提交前跑它，所有测试等于白写。fuzz / 对打基准挪进慢轨，`npm test` 只跑快轨。
- **抽公共随机数模块**：`makeLCG` / `seededShuffle` 在测试里已复制 5 份，另有两处各写各的。干瞪眼必须与掼蛋共用一个，否则两个游戏之间 seed 语义不可比，跨游戏复现 bug 的心智成本白白翻倍。

## 阶段 1：engine + 穷举单测（纯逻辑，零依赖）

`src/games/gandengyan/engine/`：`types` / `cards` / `combos` / `legal` / `wild` / `deal`。

- **牌型识别**：给定「一组牌 + 每张王的指派」，判定它是不是合法牌型、是哪种、关键点数是多少。指派非法直接拒绝。
- **歧义识别**：给定一组牌，枚举出所有**结果牌型不同**的合法指派（供客户端弹二选一）。这是 UI 与 AI 共用的入口，不是事后推断。
- **压制判定**：大一法则 + 三个逃生口 + 炸弹全序（`3 张 < 4 张 < 王炸`，同张数「大就行」，含王炸与纯炸平级，同一炸 ≤4 张）。
- **牌局状态机**：领出 / 跟牌 / 过牌 / 一轮结束 → 仅赢家摸 1 张 → 领出下一轮 / 牌堆见底 / **僵局终止** / 结算。

**验收**：`tests/gandengyan/` 覆盖 SPEC《干瞪眼验收》1–8 全部条目 + 模糊测试（随机对局必定终止、不产生非法状态）。

## 阶段 2：抽牌类公共房间层（掼蛋零回归）

把掼蛋的连接 / 昵称 / 大厅 / 建房 / 加入 / 观战 / 座位 / 房主开打 / 掉线 AI 接管 / 重连抽成与游戏无关的公共层，参数化 `{ gameId, minSeats, maxSeats }`，**不解析任何对局消息**——只按座位转发不透明 payload，游戏侧提供 `MatchRunner`。

- 掼蛋切过去（固定 4 座）；干瞪眼后续接入（2–5 座，房主建房时选目标座位数）。
- **各游戏各自大厅，昵称全服唯一**（owner 拍板）：大厅按 `gameId` 分桶，昵称占用表只有一套。
- **象棋不并入**，`server/rooms.mjs` 原样不动（ADR-0001）。

**顺序不能反**（ADR-0001 的前置条款）：先把掼蛋房间测试从「断私有字段」改造成「读只读快照 + 断出站消息」，**在旧实现上跑绿、单独 commit 冻结**，再抽层，同一份测试一字不改跑绿。跳过这步，「零回归」就是改了尺子再用它量。

**验收**：冻结后的那份测试 + `online-*.test.ts` 全绿 **+ 真机多 context 冒烟**（建房 / 加入 / 对弈 / 断线重连 / 观战）。这一步不产出新功能，只证明抽象抽对了。

---

## 阶段 3：服务端牌局驱动 + AI

- `server/gandengyan-match-driver.ts`：每个 playing 房一个权威牌局，import 阶段 1 的 engine，按座位下发；经 esbuild 打 bundle 供生产 server 用（同掼蛋做法）。
- 手牌私发：`hand` 只发本人，公开态 `state` 发全房 + 观众，别人的手牌永不进任何下发消息。
- **AI：启发式 + 记牌**。记各点数**剩余分布**（大一法则下「6 还剩几张」远比「谁手里有 A」重要）；估「我出这张下家接得上的概率」；留 2 / 炸弹当逃生口；优先拆难出的牌；王优先搭进顺子 / 对子而非做炸。空座由 AI 顶，掉线由 AI 接管、人回来收回座位。

**验收**：驱动单测（合法性校验、私发隔离、掉线接管、重连）+ AI 对打基准（弱 AI vs 强 AI 胜率，参照掼蛋 `ai-improvement.test` 打法，改 AI 看数字不拍脑袋）。

## 阶段 4：UI + 上线

- `src/games/gandengyan/ui/`：2–5 座位布局、手牌选牌、出牌 / 过牌、**歧义时浮出二选一**、轮次与剩牌数指示、结算表（把「5 张 × 2×2×2 × 2^炸数」逐项展开，别只甩一个总分）。
- 牌面复用掼蛋既有资产（Besley 字体 + Seedream 花色图 + 小丑图）。新增固定汉字重跑 `pyftsubset`。
- 壳注册表加第三个 `GameModule`，路由 `/gandengyan`。

**验收**：Playwright + 系统 Chrome（`channel:'chrome'`），**2 人局与 5 人局各打完一整局**，验赢家判定与结算数字；联机走多 context 真路径。

## 已知会咬人的点（写代码时盯住）

1. **王指派不是等价选择**——大一法则下 `王+5+6` 出成 456 还是 567 直接决定下家接不接得上。任何「自动取最强」的写法都是 bug（掼蛋 `wild.ts` 那套不能照搬）。
2. **僵局是终止性保证**，不是边角料。没有它，2 人各剩一张王就死循环——模糊测试会第一时间抓到。
3. **摸牌是独立阶段**，不是 play 的副作用：`领出 → 跟牌一圈 → 全过 → 赢家摸 → 新一轮领出`。
4. **「仅赢家摸」是追赶机制不是滚雪球**（调研笔记 §D3 的后果判断写反了）：它罚的是刚赢下一轮的领先者。AI 的手牌规划按这个真实动力学算。
5. **炸弹封顶 4 张**：`4 张 5 + 1 张王` 必须判非法，别让「张数越多越大」自然延伸出 5 张、6 张炸。
6. **僵局判定不能用 `hasAnyPlay` 验它自己**。枚举器漏解 → 僵局提前误判 → 局正常结束、步数远不到上限 → **fuzz 全绿而 bug 直达生产**。必须用暴力子集穷举当独立 oracle（手牌才 5~6 张，穷举可行）。
7. **`deck` 绝不进公开态**。掼蛋没有牌堆这个字段所以没这个雷，干瞪眼有，随手 spread 一下就泄，泄了就是完美记牌器。反过来，**桌面当前牌的 `assign` 必须进公开态**，否则重连的人看到一张王算不出大一该出什么。
8. **`assign` 是客户端可控输入**，是掼蛋没有的一整个攻击面。协议边界必须有恶意用例族。
