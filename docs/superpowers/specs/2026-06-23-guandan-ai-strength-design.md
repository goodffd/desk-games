# 掼蛋强 AI（拆牌规划 · 诚实打法）设计

> 状态：待 owner 评审 → 通过后进 writing-plans。
> 日期：2026-06-23。范围：仅掼蛋出牌 AI（`choosePlay`）+ 还贡选牌（`chooseReturn`），不动象棋、不动引擎规则。

## 问题

掼蛋的 AI（单机调试对战 + 在线真人发呆/掉线时的代打）打得很差、很机械。owner 在线看到 AI 代打"傻、被动"。

根因：现 `src/games/guandan/ai/ai.ts`（116 行，"一期基础合规"版）只有一条规则——**每次都出能出的最小一手**。导致：
1. 永远先甩最小牌、不看手里结构 → 拆散对子/三张/顺子去凑单张，最后剩一手走不掉的烂牌
2. 没有"这副牌最少几手能走完"的概念，临门也不一口气走完
3. 跟牌只会最小压制，从不为保关键牌战略性"不要"
4. 炸弹只当最后兜底，从不在该炸的时机炸
5. 配合几乎为零（只有"队友领先就不要"一条）
6. 红心2(逢人配)可能早早浪费在小牌型上
7. 还贡 `autoReturn` 取全局最小牌，可能拆掉自己的对子/结构

## 谁用 / 什么时候用

- **在线代打**（主场景）：真人掉线或回合超时，服务端 `guandan-match-driver` 用 `choosePlay` 替该座出牌。代打你自己的座位时帮你队、代打对手时也得像个会打的人。
- **单机对 AI**（`?debug`，`LocalDriver`）：同一个 `choosePlay`。
- **还贡**：AI 为收贡方时选哪张牌还回去。

`choosePlay` 是三处共用的唯一决策函数 → 改一处，三处都提升。

## 目标 / 验收

- 目标档位：**尽量强**（owner 选定），但保持快、不上搜索/MCTS。
- **诚实边界**（硬约束）：每家**手牌张数**是公开信息（掼蛋桌上谁剩几张都看得见）→ AI 可用；**手牌内容只看自己那家**，不偷看牌面、不记牌、不读出牌历史。
- 验收：
  - 新 AI vs 老 AI 对打模拟 **胜率 ≥ 60%**（owner 定），且平均每局手数下降。
  - fuzz 仍全绿（出牌合法 + 108 守恒），性能不显著拖垮（拆牌有界贪心 + 记忆化）。
  - 单测覆盖拆牌正确性 + 关键战术行为断言。
  - 真机：Playwright 逼一座超时，AI 代打打完一手牌，owner 眼验"像会打"。

## 模块边界

- **新增 `src/games/guandan/ai/decompose.ts`**（纯函数，绝不 import DOM/不另写规则）
  - `decompose(hand, level): { combos: Combo[]; handCount: number }`
  - 把自己手牌拆成**近似最少手数**的牌型组合，拿引擎 `enumerateLeads(hand, level)` 当积木。
  - 方法：枚举候选牌型 → 有界贪心 + 记忆化的递归覆盖，最小化组合数；同手数时 tie-break：① 少消耗红心2 ② 保炸弹完整 ③ 残牌点数更低。确定性（不用随机）。有界以保证 1000 局 fuzz 仍快。
- **重写 `src/games/guandan/ai/ai.ts` 的 `choosePlay`**：基于 `decompose` + 战术规则。**签名/合约不变**：`(s, seat) => Card[] | null`，永远合法、只在跟牌时返回 `null`(不要)。driver/server/UI **零改动**。
- **新增 `chooseReturn(hand, level): Card`**（在 ai/ 层）：智能还贡。driver 的 AI 座由 `autoReturn` 改调 `chooseReturn`。引擎 `match.ts/autoReturn` 保留为纯兜底（不删，避免引擎 import ai 的反向依赖）。
- 文件大小：`ai.ts` 控制在 ~250 行内，拆牌独立成文件（可单独测）。

## 决策逻辑

### 领牌（free play，`s.current === null`）
1. **能一口气走完就走完**：若手牌拆解后 1 手可清 → 出之；接近走完时朝"出完"方向走。
2. 否则领"废牌"：从拆解里挑**最低的非控制牌型**（小单张/小对子/短顺）先甩，把**控制牌**（大单张 A/2/大王、炸弹）留着管节奏。
3. **绝不拆散整牌型**去领碎牌。
4. 有等效不动红心2 的领法时，不动红心2。

### 跟牌（`s.current !== null`）
1. **队友领先**（`current.by === (seat+2)%4`）：默认不要（保留现状）。例外：本手能直接走完则出。
2. **对手领先**：在 `enumerateFollows` 里挑**最不破坏拆解结构**的最小够用牌压；
   - 若只有"拆掉关键牌型"才能压，且非残局、非关键墩 → **战略性不要**（保牌）。
3. **炸弹时机**：残局清路 / 对手剩没几张要走完时拦截 / 高价值夺回上手 才炸；用**够用的最弱炸弹**，王炸留最后；不早炸小墩。

### 红心2(逢人配)节制
拆解和决策都优先省红心2，留给炸弹/顺子/残局；能用不含红心2 的等效合法牌达成同目的就不消耗它。

### 配合（公开信息：手牌张数）
- 不抢队友上手（保留）。
- 队友手牌张数很少（快走完）时：不夺上手、领低牌给他空间。

### 还贡选牌（`chooseReturn`）
在可还牌（`returnableCards`，rankValue ≤ 10）里挑**最低的"落单"牌**——其移除不破坏拆解里的对子/三张/顺子；同条件下取点数最小。无落单可还牌时退化为当前 `autoReturn`（全局最小）。

## 数据流 / 集成

```
server/guandan-match-driver.ts ─┐
src/.../driver/local-driver.ts ─┼─ choosePlay(state, seat) → Card[] | null   (出牌/代打，签名不变)
                                └─ chooseReturn(hand, level) → Card           (AI 还贡，替换 autoReturn 调用)

ai/ai.ts ── 用 ──> ai/decompose.ts ── 用 ──> engine/legal.ts (enumerateLeads / enumerateFollows / isLegalPlay)
                                              engine/combos.ts (牌型识别，唯一规则真相)
```

引擎不反向依赖 AI；AI 不复制规则判定。

## 错误处理 / 不变量

- `choosePlay` 永远返回合法出牌或 `null`（仅跟牌时）；空手返回 `null`。
- 拆解结果须**恰好覆盖**手牌（每张用一次、不增不减）。
- 纯函数、确定性、无 DOM、无随机。
- 性能：拆解有界 + 记忆化，单次调用 ms 级；1000 局 fuzz 墙钟保持可接受（目标 < 3 分钟）。

## 非目标（YAGNI）

- 不记牌、不偷看牌面内容、不接出牌历史（诚实边界）。
- 不上搜索 / MCTS / determinized lookahead。
- 不加 AI 难度档位（单一强度）。
- 进贡选牌不做（强制出最大牌，无决策空间）；只优化还贡。
- 不动象棋、不动联机协议、不动引擎规则。
